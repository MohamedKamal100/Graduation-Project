# شرح الواجهة الأمامية (Front-End) – نظام حجز التذاكر

## الأدوات والتقنيات المستخدمة:

### 1. React 19

مكتبة JavaScript لبناء واجهات المستخدم التفاعلية باستخدام مكونات (Components).  
تُسهّل إدارة الحالة والتنقل بين الصفحات بدون إعادة تحميل الصفحة بالكامل.

### 2. Vite

أداة حديثة لبناء وتشغيل المشاريع بسرعة كبيرة أثناء التطوير، أسرع من Webpack التقليدي.

### 3. Tailwind CSS

إطار عمل CSS جاهز مبني على فكرة Utility Classes، يعني كل حاجة تقدر تتحكم فيها من خلال الكلاسات مباشرة بدون كتابة CSS يدوي كتير.

### استخدمنا مكتبات حديثة وحديثة جدًا لبناء واجهة تفاعلية وسريعة.

React وTailwind <في الاساس >
axios , reactquery >>عشان ندير البيانات من الباك إند
Formik >>عشان نحسّن تجربة المستخدم في الفورمز.
كمان ركزنا على تجربة المستخدم (UI/UX)>>Framer Motion

تمام جدًا، تعال نرتّب أفكارك على شكل نقاط مختصرة ومرتبة تقدر تبدأ بيها عرض المشروع في المناقشة، وهنقسمهم لـ 4 أقسام رئيسية:
**الفكرة – المشاكل – الحل – التقنيات**

---

### ✅ أولًا: فكرة المشروع

- مشروعنا هو **نظام حجز تذاكر الفعاليات أونلاين**.
- بيسمح للمستخدمين إنهم يحجزوا تذاكر لحفلات أو مؤتمرات إلكترونيًا.
- وفي نفس الوقت، بيوفر لأصحاب الفعاليات لوحة تحكم كاملة لإدارة الفعاليات والحجوزات.

---

### ❗ ثانيًا: المشاكل اللي بنحلها

- عدم وجود نظام حجز إلكتروني منظم وسهل الاستخدام.
- الاعتماد على الحجز اليدوي أو الورقي اللي بيؤدي لأخطاء.
- صعوبة تنظيم الفعاليات ومتابعة الحضور.
- قلة التفاعل الإلكتروني (زي الدفع أونلاين – مشاركة الفعالية – QR للتذكرة).

---

### 💡 ثالثًا: الحل اللي قدمناه

- نظام إلكتروني كامل فيه حسابات للمستخدمين والمنظمين.
- حجز تذاكر أونلاين + توليد تذكرة PDF تحتوي QR Code.
- لوحة تحكم للمنظمين لإضافة فعاليات ومتابعة الحجز.
- حماية من الحجز الزائد (Overbooking) عن طريق التحقق من العدد في قاعدة البيانات.
- تصميم تفاعلي ومتجاوب يعمل على الموبايل والكمبيوتر.
- قابل للتطوير مستقبلًا ليشمل الدفع أونلاين – إشعارات – تطبيق موبايل.

---

### 🧱 رابعًا: التقنيات المستخدمة

#### 🖥️ الواجهة الأمامية (Front-End):

- React 19 + Vite + Tailwind CSS.
- مكتبات زي:

  - axios (للتعامل مع API)
  - react-query (لتخزين البيانات والكاش)
  - formik + yup (للنماذج والتحقق)
  - framer-motion (للحركة والأنيميشن)
  - react-router-dom (للصفحات)
  - qrcode + jspdf (للتذاكر والـ QR)
  - Stripe (مدفوعات مستقبلية)

#### ⚙️ الواجهة الخلفية (Back-End):

- Laravel (PHP Framework) + MySQL.
- RESTful API + نظام تسجيل دخول بالتوكن.
- حماية بالـ Middleware + صلاحيات الأدوار.
- توليد التذاكر + تسجيل الطلبات في قاعدة البيانات.

#### 🛠️ أدوات جانبية:

- Visual Studio Code – كتابة الكود
- Postman – تجربة الـ API
- GitHub – إدارة الكود والتعاون
- Stripe – تجهيز الدفع الإلكتروني
- MySQL Workbench – إدارة قاعدة البيانات
